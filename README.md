# Week4：301563 云汉芯城章节定位

## 1. 作业目标

本项目只完成 **301563 云汉芯城招股说明书的章节定位**。当前阶段不直接生成增资、股权转让和股权结构的最终数据，而是建立一条可以审计的 human-in-the-loop 流程：

```text
原始PDF
→ 规则程序生成章节候选
→ 大模型审阅候选并提出建议
→ 人工逐页核对、调整或补充
→ 输出经人工确认的最终章节表
→ 保存自动结果与人工修改差异
```

项目重点不是证明机器可以完全替代人工，而是明确三者的职责：

- **程序规则**：读取 PDF 文本层、识别目录和标题、生成高召回章节候选；
- **大模型辅助**：根据页码化证据判断候选章节属于核心、参考还是排除，并提示图表或多事件边界；
- **人工复核**：回到原 PDF 核对页码和语义，做最终决定并留下证据备注。

## 2. 与此前方法的区别

本次代码按会议后的认识进行了重构：

1. 不再直接把整本招股说明书交给模型生成结论；
2. 不再把全文关键词命中直接当成章节结果；
3. 先检查 PDF 文本层，复杂图表页进入人工复核，不默认整本 OCR；
4. 自动结果、大模型建议、人工结论三层分开保存；
5. 人工调整不能覆盖自动结果，必须输出差异表和修改原因；
6. 最终脚本默认只接受已经逐页核对并确认的人工记录。
